# Polymarket 跟单策略 Skill

## 功能概述

本 Skill 提供完整的 Polymarket 顶级交易者追踪、分析和跟单策略生成系统。

## 核心功能

### 1. 交易者分析 (analyze)
分析 Polymarket 顶级交易者，筛选出适合追踪的目标地址。

```bash
python3 ~/.openclaw/workspace/polymarket_analyzer.py
```

**筛选标准：**
- 总盈利 > $1M
- 月度盈利 > $100K（证明近期活跃）
- 稳定性评分 > 30
- 风险评分 < 70

**输出：**
- 推荐追踪地址列表
- 每个地址的稳定性评分
- 风险评级
- 交易策略类型

### 2. 自动追踪 (track)
定期扫描目标地址的交易活动，生成跟单建议。

```bash
python3 ~/.openclaw/workspace/polymarket_tracker.py
```

**扫描频率建议：**
- 高频：每 4 小时（适合活跃交易者）
- 中频：每 6 小时（推荐）
- 低频：每 12 小时（适合长期持有者）

### 3. 信号生成 (signal)
基于多地址交叉验证生成交易信号。

**信号等级：**
- 🔴 强烈信号：≥3 个地址一致，置信度 ≥80%
- 🟡 中等信号：2 个地址一致，置信度 60-80%
- 🟢 观察信号：单个地址大额交易

## 交易策略框架

### 策略一：共识跟单策略
**逻辑：** 当 ≥2 个追踪地址在同一市场方向一致时跟进

**参数：**
- 最小共识数：2 个地址
- 最小置信度：60%
- 仓位分配：20-30%（高置信度）/ 10-15%（中置信度）
- 止损：-15%
- 止盈：+50%

### 策略二：趋势跟踪策略
**适用：** 政治事件、体育赛季等长期标的

**参数：**
- 持有周期：1-4 周
- 入场时机：多地址开始建仓时
- 加仓时机：价格回调但趋势未变
- 出场时机：出现反向信号或事件临近

### 策略三：套利策略
**适用：** 同一事件的不同市场间价差

**参数：**
- 快速进出
- 小仓位高频
- 严格止损 -5%

## 风险管理

### 仓位管理
- 单笔交易不超过总资金 30%
- 同时持仓不超过 5 个市场
- 保留 30% 现金应对机会

### 市场筛选
**排除：**
- ❌ 15 分钟内结算的短期事件
- ❌ 流动性差的市场（日交易量 <$10K）
- ❌ 新上线 < 24 小时的市场

**优选：**
- ✅ 政治事件（选举、政策）
- ✅ 体育赛季结果
- ✅ 经济指标发布
- ✅ 加密市场长期趋势

## 自动化设置

### 设置定时任务（每 6 小时）

```bash
# 编辑 crontab
crontab -e

# 添加以下行
0 */6 * * * cd ~/.openclaw/workspace && python3 polymarket_tracker.py >> tracker.log 2>&1
```

### 设置通知

可以在 tracker.py 中添加 webhook 通知：

```python
import requests

def send_notification(signal):
    webhook_url = "你的 webhook URL"
    requests.post(webhook_url, json={
        "text": f"🎯 新信号: {signal['market']} - {signal['direction']}"
    })
```

## 注意事项

⚠️ **风险提示：**
1. 过往业绩不代表未来表现
2. 跟单交易存在延迟风险
3. 建议先用小额资金测试
4. 始终保持独立判断

⚠️ **法律合规：**
- 确保你所在地区允许预测市场交易
- 遵守当地法律法规
- 注意税务申报义务
